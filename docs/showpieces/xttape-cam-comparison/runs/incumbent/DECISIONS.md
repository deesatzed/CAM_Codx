# XTtape Decisions

## 2026-06-24: Browser-First Live Showpiece

Accepted: browser-first app before desktop wrapper.

Reason: fastest route to live proof, screenshots, and novice-friendly video.

## 2026-06-24: FastAPI Plus Node/Prisma

Accepted: FastAPI app brain with Node/Prisma data service.

Reason: keeps AI/source orchestration separate from durable product storage.

## 2026-06-24: Grok/xAI First

Accepted: Grok/xAI is the first AI provider.

Reason: user has an `XAI_API_KEY` ready locally and wants X/social intelligence to be central to the app.

## 2026-06-24: Custom Read-Only X MCP

Accepted: own narrow X MCP informed by X Agent Resources and `xdevplatform/xmcp`.

Reason: avoids broad write-capable tool exposure and keeps the app source-backed.

## 2026-06-24: CAM_Codx Guides The Build Only

Accepted: use CAM_Codx/claw.db/configs for build-process memory, not XTtape runtime data.

Reason: preserves the user's original constraint: use existing CAM context so the showpiece covers the build process, not CAM setup.
