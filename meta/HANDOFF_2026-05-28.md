## 4. CAM_CAM Changes (session 1, 2026-05-29)

### New: `cam learn ingest-codex-outcomes`

Added to `src/claw/cli/_monolith.py`. Consumes rows from `codex_outcome_log` (the MCP outcome staging table) and routes them into the CAM_CAM corpus via `semantic_memory.record_outcome()`.

**Usage:**
```bash
# See pending rows without writing
cam learn ingest-codex-outcomes --dry-run

# Ingest all pending rows
cam learn ingest-codex-outcomes

# Ingest from a specific outcome DB
cam learn ingest-codex-outcomes --outcome-db /path/to/codex_outcome_log.db

# Verbose: show each row as it's processed
cam learn ingest-codex-outcomes -v
```

**Idempotency:** processed row IDs tracked in `codex_outcome_ingested` table in `claw.db`. Re-runs are safe.

**Outcome mapping:** `green` → `success=True`; `red`/`partial`/`rejected` → `success=False`.

**Tests:** `tests/test_learn_ingest_codex_outcomes.py` — 9 tests, all pass.

### Fix: Verifier drift alignment + LLM deep review (commit `e10e298`)

**Root cause:** Verifier was using corpus embedding engine (`hash-embedding-384`) for drift alignment checks. Hash vectors have no semantic meaning — cosine similarities were random (often negative), causing all `cam enhance` runs to be rejected with `drift_alignment: CRITICAL`.

**Fix 1 — `src/claw/core/factory.py`:**
The factory now creates a separate `EmbeddingEngine` with `EmbeddingsConfig()` (default `all-MiniLM-L6-v2`) for the verifier, bypassing the `required_model = "hash-embedding-384"` constraint.

**Fix 2 — `src/claw/verifier.py`:**
The `_DEFAULT_DEEP_CHECK_PROMPT` was missing `{task_description}`, `{approach_summary}`, and `{diff}` substitution placeholders, so LLM received empty context and returned "no diff provided." Added all three placeholders.

**Verification:** Tested `cam enhance /Volumes/WS4TB/repo421sn/autoresearch` → `approved=True, violations=0, quality=1.00`. Second run triggered 3 embryonic→viable transitions.

**Tests:** 4,227 passed, 2 pre-existing failures unchanged (test_preflight_cli, test_serial_evolution model config).
