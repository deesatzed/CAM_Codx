## 3. CAM_CAM Changes (this session)

### New: `cam learn ingest-codex-outcomes`

Added to `src/claw/cli/_monolith.py`. Consumes rows from `codex_outcome_log` (the MCP outcome staging table) and routes them into the CAM_CAM corpus via `semantic_memory.record_outcome()`.

**Usage:**
```bash
# See pending rows without writing
cam learn ingest-codex-outcomes --dry-run

# Ingest all pending rows
cam learn ingest-codex-outcomes

# Ingest from a specific outcome DB
cam learn ingest-codex-outcomes --outcome-db /path/to/codex_outcome_log.db

# Verbose: show each row as it's processed
cam learn ingest-codex-outcomes -v
```

**Idempotency:** processed row IDs tracked in `codex_outcome_ingested` table in `claw.db`. Re-runs are safe.

**Outcome mapping:** `green` → `success=True`; `red`/`partial`/`rejected` → `success=False`.

**Tests:** `tests/test_learn_ingest_codex_outcomes.py` — 9 tests, all pass.
