## What This Project Does

`CAM_Codx` is a workspace directory containing two things:

1. **`CAM_CAM/`** — a substantial, working Python research engine (claw v0.1.0). Mines methodologies from real GitHub repos, scores them with a bandit, runs a 3-layer defense chain to rescue failed builds, exports knowledge packs. Heavy. Run out-of-band.
2. **`.codex/`** — an OpenAI Codex CLI install. 38 skills, 10 agents, an AGENTS.md doctrine, 80+ trusted project entries. Markdown-first workflow.

**The brainstorm in progress** is designing a *new methodology* that lets Codex consume selected CAM_CAM superpowers (pattern recall with provenance, fitness-tracked outcomes, cross-repo decision search) **without running CAM_CAM inline**. The bridge is a purpose-built thin MCP server (4 tools, hard ceiling) that sits between the two planes.

**Tech Stack:** Python 3.13 (CAM_CAM requires ≥3.12), SQLite + FTS5 + sqlite-vec, official `mcp` SDK (stdio transport), pydantic v2. Codex CLI uses `gpt-5.5` with `high` reasoning effort.

**Architecture Pattern:** Three planes (see Section 1 of design below): Codex orchestrator ↔ thin librarian MCP ↔ heavy CAM_CAM engine (out-of-band).

## The Design (state as of pause)

### Section 1 — Architecture & Doctrine (PRESENTED, AWAITING APPROVAL)

**Working name:** Codex-CAM Methodology
**Thesis:** CAM_CAM stays heavy and out-of-band. Codex stays markdown-native. A thin purpose-built MCP sits between them as a *librarian, not a co-pilot*.

**Three planes:**
1. **Codex** — orchestrator. Native skills, agents, AGENTS.md. Reads/writes GOAL/STANDARDS/IMPLEMENT/DECISIONS/PROGRESS.
2. **CAM-Codex MCP** — thin librarian. 4 tools max. Read-mostly facade over `claw.db`. Append-only writes to fitness ledger. Hard ceiling enforced in CI.
3. **CAM_CAM** — heavy engine. Mining, bandit, evolution, federation, dashboards. Runs out-of-band. Never invoked inline.

**Boundary rule (the single decision everything follows from):**
> Stateful + cross-repo + computational → MCP.
> Doctrine + workflow + output schema → Skill.
> Anything that fits in markdown → Markdown.

**Doctrine additions to `.codex/AGENTS.md`:**
> CAM_CAM is consulted as a librarian via MCP, never run inline.
> - No mined methodology may be applied without its provenance row written to `IMPLEMENT.md`.
> - On second consecutive verification failure, the `rescue_ladder` skill runs before the user is asked.
> - After any verified step that used a recalled methodology, `outcome_log` must record the result.
> - "Codex decides. Claude contributes. Tests arbitrate. Markdown remembers. CAM librarian cites."

**Explicit rejections (v1):** no daemons; no model-dispatch tools; no write tool besides `cam_record_outcome`; no silent pattern application above any fitness threshold; no "889 patterns" marketing claim (we have 107).
